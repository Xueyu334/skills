# I18n 国际化

NocoBase 插件支持前后端统一的多语言国际化。插件文案 交互提示 接口返回信息和 CLI 提示应尽量通过 i18n 管理，而不是硬编码。

## 语言文件目录

插件语言文件统一放在 `src/locale`：

```text
plugin-name/
└─ src
   └─ locale
      ├─ en-US.json
      └─ zh-CN.json
```

示例：

```json
{
  "Hello": "你好",
  "World": "世界",
  "Enter your name": "请输入你的名字",
  "Your name is {{name}}": "你的名字是 {{name}}"
}
```

约定：

- 文件按语言命名，例如 `zh-CN.json` `en-US.json`
- 文案使用统一 key
- 首次新增语言文件后通常需要重启应用才能生效

可通过接口校验语言是否生效：

```text
http://localhost:13000/api/app:getLang?locale=zh-CN
```

## 全局 i18n

`app.i18n` 适合 CLI 或全局场景：

```ts
import select from '@inquirer/select';
import input from '@inquirer/input';

export class PluginSampleI18nServer extends Plugin {
  load() {
    this.app.command('test-i18n').action(async () => {
      const answer1 = await select({
        message: 'Select a language',
        choices: [
          { name: '中文', value: 'zh-CN' },
          { name: 'English', value: 'en-US' },
        ],
      });

      await this.app.changeLanguage(answer1);

      const answer2 = await input({
        message: this.app.i18n.t('Enter your name'),
      });

      console.log(this.app.i18n.t('Your name is {{name}}', { name: answer2 }));
    });
  }
}
```

常用调用：

- `app.i18n.t(text, options)`
- `app.t(text, options)`

## 请求上下文 i18n

每个请求都有自己的 `ctx.i18n` 克隆实例，会根据客户端语言独立响应。

客户端指定语言的常见方式：

- Query String：`?locale=en-US`
- Header：`X-Locale: en-US`

中间件示例：

```ts
export class PluginSampleI18nServer extends Plugin {
  load() {
    this.app.use(async (ctx, next) => {
      if (ctx.path === '/api/test-i18n') {
        ctx.body = ctx.t('Hello', { ns: '@my-project/plugin-hello' });
      }
      await next();
    });
  }
}
```

常用调用：

- `ctx.i18n`
- `ctx.t(text, options)`

## 插件内部 i18n

插件内部可以直接使用 `plugin.t()`：

```ts
export class PluginSampleI18nServer extends Plugin {
  load() {
    this.app.use(async (ctx, next) => {
      if (ctx.path === '/api/plugin-i18n') {
        ctx.body = this.plugin.t('Hello');
      }
      await next();
    });
  }
}
```

约定：

- `plugin.t(text)` 可视作带插件命名空间的翻译调用
- 服务端响应文案优先使用 `ctx.t()` 或 `plugin.t()`，不要直接硬编码

## 相关 API

- `app.i18n`
- `app.t(text, options)`
- `ctx.i18n`
- `ctx.t(text, options)`
- `plugin.t()`
- `tExpr(text, options)`

## 实践建议

- 用户可见文案统一落到 `src/locale`，不要散落在 handler middleware 组件里。
- 需要按请求语言返回文案时优先使用 `ctx.t()`。
- 插件内部固定命名空间文案优先使用 `plugin.t()`。
- CLI 或全局逻辑使用 `app.i18n`。
